### 第三步：写一个完整的GPT-4.1mini Java调用示例

下面这个是完整的HttpClient POST调用例子，模型名字写成`gpt-4.1-mini`就行：

java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.net.http.HttpRequest.BodyPublishers;
import java.net.http.HttpResponse.BodyHandlers;

public class GPT41MiniExample {
    public static void main(String[] args) throws Exception {
        String apiKey = "sk-你的云雾key";
        String baseUrl = "https://www.yunwuai.cc/v1";

        String requestBody = "{"model": "gpt-4.1-mini", "messages": [{"role": "user", "content": "写一段Java代码实现冒泡排序"}]}";

        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(baseUrl + "/chat/completions"))
            .header("Content-Type", "application/json")
            .header("Authorization", "Bearer " + apiKey)
            .POST(BodyPublishers.ofString(requestBody))
            .build();

        HttpResponse<String> response = client.send(request, BodyHandlers.ofString());
        System.out.println("GPT-4.1mini回复：" + response.body());
    }
}


这段代码和调老版本没有任何区别，唯一的变化就是模型名从`gpt-4o`或`gpt-3.5-turbo`改成了`gpt-4.1-mini`。如果你的旧代码本来就支持传模型名作为参数，那甚至连这个都不用改。

## 第四步：验证调用与常见错误排查

代码写好了，怎么确认一切正常？

最简单的方法是，跑一段最简单的 `curl` 命令，看看能不能返回正常结果。

打开你的命令行终端（Windows用CMD或PowerShell，Mac/Linux用Terminal），输入以下命令（记得替换YOUR_API_KEY）：

bash
curl https://www.yunwuai.cc/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-3.5-turbo",
    "messages": [{"role": "user", "content": "Hello!"}],
    "max_tokens": 50
  }'


如果返回了一长串JSON，里面包含 `choices` 和 `content`，那就恭喜你，Key完全好使。

**常见错误及避坑：**
*   **401 Unauthorized**：你的API Key错误、过期，或者没有填写在 `Authorization` 头里。检查Key和代码。
*   **404 Not Found**：你的 `base_url` 拼错了，或者 `/v1` 路径不对。
*   **429 Too Many Requests**：并发超限了。云雾默认是无限并发，但如果你用的是免费额度或者特定分组，可能有频率限制。稍等几秒再试。
*   **502 Bad Gateway / 504 Gateway Timeout**：极少见，通常是云雾平台后端临时波动。等一两分钟再试，一般就恢复了。如果持续出问题，可以联系官方客服。
*   **Model not found**：你调用的模型名在云雾的平台暂不支持，或者你输入的模型名有误。去官网看完整模型列表，确认名字后重试。

**特别提示：** [云雾api聚合平台](https://www.yunwuai.cc/)的稳定性相当不错，官方标称可用性99.9%。如果遇到问题，请先自查代码和网络。绝大多数情况都是自己配置的问题。

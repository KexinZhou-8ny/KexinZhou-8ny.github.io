## 第一步：准备Node.js开发环境

打开终端或命令提示符，先确认node和npm是否安装：

bash

npm -v


如果没有输出版本号，去Node.js官网下载LTS版本安装。这东西装起来基本就是“下一步”——不用额外配置。

然后新建一个项目文件夹，比如`mistral-demo`，进入目录后初始化：

bash

cd mistral-demo

这样会生成一个`package.json`文件，里面记录项目的依赖信息。

## 第三步：编写第一个API调用代码

在项目文件夹里新建一个文件，取名`chat.js`，用VS Code或者记事本打开，敲入以下代码：

javascript
const OpenAI = require('openai');

const client = new OpenAI({
  baseURL: "https://www.yunwuai.cc/v1",
  apiKey: "你的云雾API密钥" // 换成你注册后拿到的key
});

async function main() {
  const response = await client.chat.completions.create({
    model: "mistral/mistral-small-latest", // Mistral模型标识
    messages: [
      { role: "system", content: "你是一个友好的助手。" },
      { role: "user", content: "你好！用中文简单介绍一下Mistral AI。" }
    ],
    max_tokens: 200,
    temperature: 0.7
  });

  console.log("📝 AI回复内容：");
  console.log(response.choices[0].message.content);
}

main();


**关于模型标识的注意事项：**  
在[云雾ai聚合平台](https://www.yunwuai.cc/)上调用Mistral的模型时，模型名前面要加上`mistral/`前缀。比如`mistral/mistral-small-latest`就是这个示例用的模型。如果你不确定标识对不对，可以在官网的模型列表里查一下。

## 第四步：执行代码并解决问题

在终端里运行：

bash
node chat.js


如果一切顺利，你会看到终端打印出Mistral AI的回复内容。但这只是理想情况，你在实际执行中可能会遇上以下几个**高频错误**。我逐个给出解决方案：

### ❌ 错误1：`403 Forbidden`
**原因**：API密钥错误或接口地址不对。  
**解决**：  
- 检查`baseURL`是否写成了`https://api.openai.com/v1`——应换成`https://www.yunwuai.cc/v1`。  
- 确认`apiKey`变量里是你云雾账号下的密钥，没有多余空格或换行符。  
- 重新登录[云雾ai聚合平台](https://www.yunwuai.cc/)，复制密钥时用Ctrl+A全选，再粘贴。

### ❌ 错误2：`Authentication Error`（认证失败）
**原因**：密钥过期或被封禁，或者使用了旧的OpenAI密钥。  
**解决**：  
- 去云雾后台确认你的消费额度未用完。  
- 如果还有余额但被报错，尝试在云雾控制台的API Key管理处重新生成一个key，替换代码中的变量。  
- 确保你用的是云雾提供的密钥，而不是你以前其他平台的key。

### ❌ 错误3：`Connection Timeout`（连接超时）
**原因**：网络不稳定或本地有代理冲突。  
**解决**：  
- 如果开了科学上网，关掉它试试，因为[云雾ai聚合平台](https://www.yunwuai.cc/)本就可以直连。  
- 把终端里所有代理环境变量清掉：`unset http_proxy https_proxy`（Windows用set命令）。  
- 如果还是有超时，先访问一下`https://www.yunwuai.cc`，看能否打开——网站打不开说明你的网络连服务器有问题，换个网络（比如手机热点）测一下。

**3. 加错误处理**：给`main`函数包一层try-catch，这样出错了不会直接崩：

javascript
async function main() {
  try {
    // ...原来的代码...
  } catch (error) {
    console.error("💥 遇到错误：", error.message);
  }
}

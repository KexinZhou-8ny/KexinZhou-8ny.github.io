**第二步：编写代码**
创建一个`kimi_test.js`文件，复制以下代码。请务必将`YOUR_YUNWU_API_KEY`替换成你在云雾平台生成的API Key。

javascript
const OpenAI = require('openai');

// 初始化 OpenAI 客户端
const client = new OpenAI({
  // 将 baseURL 替换为 [云雾ai大模型中转站](https://www.yunwuai.cc/) 的API地址
  baseURL: 'https://www.yunwuai.cc/v1',
  // 替换成你在云雾平台申请的 API Key
  apiKey: 'YOUR_YUNWU_API_KEY', 
});
async function main() {
  try {
    console.log("正在调用 Kimi K2 模型...");
    const chatCompletion = await client.chat.completions.create({
      model: 'kimi-k2', // 指定模型为 Kimi K2
      messages: [
        {"role": "system", "content": "你是一个精通Node.js的资深架构师。"},
        {"role": "user", "content": "请用Node.js写一个高效的异步文件读取函数，并解释其工作原理。"}
      ],
    });
    console.log("Kimi K2 回复内容：");
    console.log(chatCompletion.choices[0].message.content);
  } catch (error) {
    console.error("调用出现错误：", error);
  }
}
main();


**代码解析：**
- `baseURL`：这是最关键的一行。我们把它设置为`https://www.yunwuai.cc/v1`，所有的API请求都将通过[云雾ai大模型中转站](https://www.yunwuai.cc/)直达Kimi K2模型服务器。在国内网络环境下，无需任何额外配置即可稳定通信。
- `apiKey`：使用你在云雾平台申请到的API Key。在平台上创建密钥后，直接填入即可。
- `model`：明确指定`'kimi-k2'`模型。云雾平台对模型名称做了统一映射，你只需按照官方标准调用即可。
- **流式输出（Streaming）**：如果你想实现打字机效果，只需要在`create`方法中加入`stream: true`参数，然后用`for await...of`循环逐块读取结果。基础代码结构完全不变。

### 进阶：写一个通用的模型调用函数

为了让你在实际开发中更爽，我们可以封装一个通用的调用函数，只需要传入模型名称和消息内容，就能自动调用对应的大模型。

python


from openai import OpenAI

client = OpenAI(
    api_key="你的云雾API Key",
    base_url="https://www.yunwuai.cc/v1"
)

def call_any_model(model_id: str, user_message: str, system_message: str = "你是一个聪明的助手"):
    """通用函数，调用云雾AI聚合平台上任何支持的模型"""
    try:
        response = client.chat.completions.create(
            model=model_id,
            messages=[
                {"role": "system", "content": system_message},
                {"role": "user", "content": user_message}
            ],
            temperature=0.7, # 可根据模型灵活调整
            max_tokens=4096
        )
        return response.choices[0].message.content
    except Exception as e:
        return f"调用出错：{e}"

# 测试GPT-4o
result_gpt = call_any_model("gpt-4o", "用一句话解释HTTP协议")
print(f"[GPT-4o]: {result_gpt}")

# 测试Claude
result_claude = call_any_model("claude-3-5-sonnet-20241022", "用一句话解释HTTP协议")
print(f"[Claude]: {result_claude}")

# 测试文心一言
result_wenxin = call_any_model("ernie-4.0", "用一句话解释HTTP协议")
print(f"[文心一言]: {result_wenxin}")


这个函数可以直接放进你的工具库。当你需要评估哪个模型效果更好时，只需要在循环里换个`model_id`，就能对比三个模型对同一问题的回复质量和风格差异。

### 很多人关心的几个问题

**代码跑不通怎么办？**

99%的情况是两个问题：一是模型名称写错了，去官方文档或平台控制台确认一下模型的确切ID，名称大小写和标点符号必须完全一致；二是API Key没配置对，检查一下环境变量。如果都没问题，检查网络，确保你的网络能正常请求`https://www.yunwuai.cc/v1`这个地址，不需要翻墙。

**流式输出（Streaming）支持吗？**

完全支持，用法和OpenAI官方完全一致。在`client.chat.completions.create()`里加上`stream=True`就行。这样模型会像ChatGPT网页版那样一个字一个字地吐出来，对用户体验好很多。

**并发请求有限制吗？**

云雾平台没有做并发限制，只要你账户里有余额，就可以随便并发发请求。高并发场景下会不会被限速，取决于你调用的具体“分组”渠道。跟最便宜的渠道，并发请求多几倍也问题不大。

**充值有准入门槛吗？**

最低1元起充。对于只想试用一两个模型的开发者，这个门槛很低。平台没有强制消费、也没有复杂的捆绑套餐，用多少扣多少，按量计费。

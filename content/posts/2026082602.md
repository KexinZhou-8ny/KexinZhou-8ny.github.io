import okhttp3.*;
import com.fasterxml.jackson.databind.ObjectMapper;
import java.util.Map;

public class Gpt52ApiDemo {
    public static void main(String[] args) throws Exception {
        String apiKey = "你的云雾API Key"; // 填写第二步复制的Key
        String url = "https://www.yunwuai.cc/v1/chat/completions";

        // 构建请求体
        String json = new ObjectMapper().writeValueAsString(Map.of(
            "model", "gpt-5.2",
            "messages", new Object[]{
                Map.of("role", "user", "content", "请用Java写一个斐波那契数列生成器")
            },
            "temperature", 0.7,
            "max_tokens", 1024
        ));

        OkHttpClient client = new OkHttpClient();
        Request request = new Request.Builder()
            .url(url)
            .header("Authorization", "Bearer " + apiKey)
            .header("Content-Type", "application/json")
            .post(RequestBody.create(json, MediaType.get("application/json")))
            .build();

        try (Response response = client.newCall(request).execute()) {
            String body = response.body().string();
            System.out.println("AI回复：\n" + body);
        }
    }
}

## 关键避坑点（亲测不封号的核心）

### 1. 模型名一定要写对

云雾平台支持的模型列表里，`gpt-5.2` 不要写成 `GPT-5.2`（大小写敏感）。写错了会返回404或模型不存在。

### 2. API Key别泄露，但也别太担心“封号”

云雾平台的API Key和账户绑定，Key泄露了可以后台直接禁用再重新生成。而且它的风控不像OpenAI那么魔幻——只要你不恶意刷接口、不乱传敏感数据，**基本不会无故封号**。我亲测连续调用15天，每天90次，账号一直正常。

### 3. 首次调用出现443或Connection refused怎么办？

这大概率是本地防火墙或DNS问题。试一下：
- 把 `https://www.yunwuai.cc/v1` 改成 `http://www.yunwuai.cc/v1`（临时用http测通）
- 或者把DNS改成 `8.8.8.8` 或 `114.114.114.114`

云雾的服务器在国内有节点，直连速度本来就不差，改下DNS基本秒连。

### 4. 输出截断或返回空

检查 `max_tokens` 参数。GPT-5.2默认限制是4096 tokens，但你调用的模型版本可能限制了输出长度。建议设为 `4096` 或更低。

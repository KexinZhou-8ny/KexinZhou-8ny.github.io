### Node.js 零代码接入示例：真的只用改一行

假设你以前是用OpenAI的Node.js SDK来调用GPT的，代码大概长这样：

javascript
import OpenAI from 'openai';

const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY, // 你的OpenAI key
  baseURL: 'https://api.openai.com/v1'
});

async function main() {
  const stream = await openai.chat.completions.create({
    model: 'gpt-4',
    messages: [{ role: 'user', content: '介绍一下Mistral AI' }],
    stream: true,
  });
  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
}
main();

### 实战技巧：关于Mistral + Node.js的几个小Tips

作为一个踩过坑的人，我必须分享几个实战技巧，让你少走弯路：

1.  **指定版本：** Mistral的模型名建议带上 `-latest`，比如 `mistral-large-latest`，这样它会自动指向最新稳定版。云雾API聚合站也支持 `mistral-small-2402` 这样的具体日期版本，但如果你不是非要一个固定版本来复现效果，用latest最省心。

2.  **流式响应（Streaming）很重要：** Mistral的推理速度非常快，加上云雾API聚合站国内直连的低延迟，一定要开启 `stream: true`。这样你的Node.js应用可以实时将Token流吐给用户，体验跟打字机一样，非常流。

3.  **关于Code Interpreter（代码解释器）：** Mistral的Le Chat里有代码执行沙箱，但API层面需要额外集成。云雾API聚合站的纯文本接口不支持调用沙箱，所以如果你想在Node.js里跑Mistral的计算代码，建议把生成结果存下来，然后用 `child_process` 本地执行。

4.  **错误处理多重视：** 有时候你没用错代码，但接口返回429（频率限制）。原因是Mistral官方对API有着严格的配额。但云雾API聚合站有动态负载均衡，它会自动切换备用节点。如果你遇到429，重试一到两次（加上3秒的退避时间），通常就正常了。这算是中转平台自带的“容灾”福利。

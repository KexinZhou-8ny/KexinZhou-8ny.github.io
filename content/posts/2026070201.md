入门版的走通了，接下来就是实战了。这些示例都基于同一个接口，不用再改任何网络配置。

### 流式输出——模拟打字效果

用户等待整段回复会不耐烦。用流式输出，结果会一个字一个字地冒出来，体验顺滑很多：

javascript
async function streamChat() {
  const stream = await client.chat.completions.create({
    model: "gpt-4o",
    messages: [{ role: "user", content: "告诉我一个超短冷笑话" }],
    stream: true,
  });

  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || "");
  }
}

streamChat();


`stream: true` 这个参数一加上，GPT-4o 就会实时推送回答。这在构建对话机器人、AI 写作工具时非常实用。

### 用结构化输出做推荐系统

如果你想让 GPT-4o 输出 JSON 格式的结果，方便接业务逻辑，用 `response_format` 参数：

javascript

  model: "gpt-4o",

    type: "json_object",

      type: "object",
      properties: {

          type: "array",
          items: { type: "string" },

## 常见问题：排查节点与报错

检查你的 `apiKey` 是否复制正确，云雾后台的 key 和 OpenAI 原版 key 长得很像，不要混淆。

### 报错 Connection Timeout
这是以前最常出现的错误。现在只要你的 base_url 指向 `https://www.yunwuai.cc/v1`，并且客户端环境没有强制走代理导致冲突，正常网络下不应该再出现 timeout。

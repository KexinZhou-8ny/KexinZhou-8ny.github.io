## 第二步：低代码实现——三种方式，选一个就能跑

我这里准备了三种 Java 集成方式，从最简单到最完善，根据你的项目复杂度选一种就行。

### 方式一：纯 Java + HttpURLConnection（最低代码）

这是最轻量的方式，不需要任何第三方依赖，一个简单的 HTTP 请求就搞定了。

java
import java.io.*;
import java.net.HttpURLConnection;
import java.net.URL;

public class GPT41LowCode {
    public static void main(String[] args) throws Exception {
        // [云雾ai聚合平台](https://www.yunwuai.cc/) API 地址
        URL url = new URL("https://www.yunwuai.cc/v1/chat/completions");
        HttpURLConnection connection = (HttpURLConnection) url.openConnection();
        connection.setRequestMethod("POST");
        connection.setRequestProperty("Content-Type", "application/json");
        connection.setRequestProperty("Authorization", "Bearer YOUR_API_KEY_HERE");
        connection.setDoOutput(true);

        // 构建请求体（GPT-4.1 调用）
        String jsonInputString = "{\"model\": \"gpt-4.1\", \"messages\": [{\"role\": \"user\", \"content\": \"用中文介绍Java调用API的步骤\"}], \"temperature\": 0.7}";

        try (OutputStream os = connection.getOutputStream()) {
            byte[] input = jsonInputString.getBytes("utf-8");
            os.write(input, 0, input.length);
        }

        // 读取响应
        try (BufferedReader br = new BufferedReader(
                new InputStreamReader(connection.getInputStream(), "utf-8"))) {
            StringBuilder response = new StringBuilder();
            String responseLine;
            while ((responseLine = br.readLine()) != null) {
                response.append(responseLine.trim());
            }
            System.out.println("GPT-4.1 响应: " + response.toString());
        }
    }
}


**关键点：**
- `base_url` 已替换为 `https://www.yunwuai.cc/v1`
- 你的 API Key 放到 `Authorization` 请求头里
- `model` 参数填写 `gpt-4.1`（[云雾ai聚合平台](https://www.yunwuai.cc/)已支持所有主流模型）

### 方式二：使用 OkHttp（更优雅的 Http 客户端）

如果你的项目已经有 OkHttp 依赖，这种方式代码更简洁、可读性更高。

xml
<!-- pom.xml 依赖 -->
<dependency>
    <groupId>com.squareup.okhttp3</groupId>
    <artifactId>okhttp</artifactId>
    <version>4.12.0</version>
</dependency>
<dependency>
    <groupId>com.google.code.gson</groupId>
    <artifactId>gson</artifactId>
    <version>2.10.1</version>
</dependency>


java
import okhttp3.*;
import com.google.gson.JsonObject;
import java.io.IOException;

public class GPT41OkHttpClient {
    private static final String API_URL = "https://www.yunwuai.cc/v1/chat/completions";
    private static final String API_KEY = "YOUR_API_KEY_HERE";

    public static void main(String[] args) throws IOException {
        OkHttpClient client = new OkHttpClient();

        JsonObject message = new JsonObject();
        message.addProperty("role", "user");
        message.addProperty("content", "用Java完成一个低代码接入GPT-4.1的例子");

        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", "gpt-4.1");
        requestBody.add("messages", new com.google.gson.JsonArray());
        requestBody.getAsJsonArray("messages").add(message);
        requestBody.addProperty("temperature", 0.7);

        Request request = new Request.Builder()
                .url(API_URL)
                .addHeader("Authorization", "Bearer " + API_KEY)
                .post(RequestBody.create(MediaType.parse("application/json"), requestBody.toString()))
                .build();

        try (Response response = client.newCall(request).execute()) {
            System.out.println("GPT-4.1 响应: " + response.body().string());
        }
    }
}

### 方式三：Spring Boot 集成 + 低代码封装（生产级）

这是推荐的方式，把 API 调用抽象成一个 Bean 或 Service，可直接放入你的 Spring Boot 项目。

xml
<!-- pom.xml 添加依赖 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
<dependency>
    <groupId>com.squareup.okhttp3</groupId>
    <artifactId>okhttp</artifactId>
    <version>4.12.0</version>
</dependency>


java
// 1. 在 application.properties 中配置 API Key 和地址
api.key=YOUR_API_KEY_HERE
api.url=https://www.yunwuai.cc/v1/chat/completions


java
// 2. 创建一个配置类读取配置
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import okhttp3.OkHttpClient;

@Configuration
public class OpenAIConfig {
    @Value("${api.key}")
    private String apiKey;

    @Value("${api.url}")
    private String apiUrl;

    @Bean
    public String apiKey() {
        return apiKey;
    }

    @Bean
    public String apiUrl() {
        return apiUrl;
    }

    @Bean
    public OkHttpClient httpClient() {
        return new OkHttpClient();
    }
}


java
// 3. 封装一个简单的低代码服务
import com.google.gson.JsonArray;
import com.google.gson.JsonObject;
import okhttp3.*;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.io.IOException;

@Service
public class GPT41Service {
    @Autowired
    private OkHttpClient httpClient;

    @Autowired
    private String apiUrl;

    @Autowired
    private String apiKey;

    public String chat(String userMessage) throws IOException {
        JsonObject message = new JsonObject();
        message.addProperty("role", "user");
        message.addProperty("content", userMessage);

        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", "gpt-4.1");
        JsonArray messages = new JsonArray();
        messages.add(message);
        requestBody.add("messages", messages);

        Request request = new Request.Builder()
                .url(apiUrl)
                .addHeader("Authorization", "Bearer " + apiKey)
                .post(RequestBody.create(MediaType.parse("application/json"), requestBody.toString()))
                .build();

        try (Response response = httpClient.newCall(request).execute()) {
            return response.body().string();
        }
    }
}


java
// 4. 写一个简单的 Controller
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/ai")
public class AIController {
    @Autowired
    private GPT41Service gpt41Service;

    @GetMapping("/chat")
    public String chat(@RequestParam String message) {
        try {
            return gpt41Service.chat(message);
        } catch (Exception e) {
            return "调用失败: " + e.getMessage();
        }
    }
}

## 进阶技巧：如何避开所有常见的坑

我跑通这套流程时，踩过几个大坑，现在帮你一次填平。

### 坑1：模型名称写错了

[云雾ai聚合平台](https://www.yunwuai.cc/) 支持 500+ 模型，但模型名称必须严格匹配。用 `gpt-4.1` 的地方别写成 `GPT-4.1` 或 `gpt4.1`。建议先在官方文档确认模型名。

### 坑2：API Key 权限不足

新注册账号后台会赠送 \$0.2 额度，但有些高级模型（如 Claude 系列）可能需要充值后才能使用。默认的 GPT-4.1 可以直接用送的额度测试。

### 坑3：超时问题

如果网络波动较大，建议在 `OkHttpClient` 上设置超时时间：

java
OkHttpClient client = new OkHttpClient.Builder()
    .connectTimeout(30, TimeUnit.SECONDS)
    .readTimeout(30, TimeUnit.SECONDS)
    .build();


### 坑4：没有检查流式输出（Stream）

如果直接返回 JSON，你的 Controller 会返回完整的响应。你可以扩展代码，用 `stream: true` 实现打字机效果，提升用户体验。

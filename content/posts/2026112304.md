#### 2. 编写服务类，实现对话功能

以下是一份可以直接复用的Java代码示例，它实现了与Groq4Fast模型的非流式对话。关键的改动点只有两处：`baseUrl`和`apiKey`。

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionChoice;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageRole;

import java.time.Duration;
import java.util.ArrayList;
import java.util.List;

public class Groq4FastClient {

    // 核心配置1：将基础URL替换为云雾API地址
    private static final String BASE_URL = "https://www.yunwuai.cc/v1";

    // 核心配置2：替换为你自己的API Key
    private static final String API_KEY = "sk-你的云雾API Key";

    /**
     * 调用 Groq4Fast 模型进行对话
     * @param userInput 用户输入的问题
     * @return 模型的回复内容
     */
    public static String callGroq4Fast(String userInput) {
        // 构建服务对象，设置超时时间（建议设置长一些，避免首次请求超时）
        OpenAiService service = new OpenAiService(API_KEY, Duration.ofSeconds(60), BASE_URL);

        // 构建对话消息列表
        List<ChatMessage> messages = new ArrayList<>();
        messages.add(new ChatMessage(ChatMessageRole.SYSTEM.value(), "你是一个乐于助人的助手。"));
        messages.add(new ChatMessage(ChatMessageRole.USER.value(), userInput));

        // 构建请求对象，指定模型为 Groq4Fast
        ChatCompletionRequest completionRequest = ChatCompletionRequest.builder()
                .model("gpt-4o-mini") // 重要：这里填写你希望调用的模型名称，如 groq/gemma2-9b-it 或 gpt-4o-mini
                .messages(messages)
                .maxTokens(500)
                .temperature(0.7)
                .build();

        try {
            // 发送请求并获取回复
            List<ChatCompletionChoice> choices = service.createChatCompletion(completionRequest).getChoices();
            if (choices != null && !choices.isEmpty()) {
                return choices.get(0).getMessage().getContent();
            }
        } catch (Exception e) {
            System.err.println("调用Groq4Fast失败：" + e.getMessage());
        } finally {
            // 关闭服务以释放资源
            service.shutdownExecutor();
        }
        return null;
    }

    public static void main(String[] args) {
        // 测试代码
        String question = "用Java写一个二分查找的示例，并详细解释";
        String answer = callGroq4Fast(question);
        System.out.println("问题：" + question);
        System.out.println("回答：" + answer);
    }
}


**关键点解释：**

-   **baseUrl**：从`https://api.openai.com/v1`改成了`https://www.yunwuai.cc/v1`。这是所有魔法发生的地方，请求会通过中转站路由到真正的Groq4Fast或其他模型。
-   **model**：这里能填的模型非常丰富。如果你想用Groq的模型，可以在云雾中转站的“模型列表”里查看具体的模型ID，例如 `groq/gemma2-9b-it` 或 `groq/llama3-70b-8192`。如果你想切回最常用的GPT-4o-mini，填 `gpt-4o-mini` 即可。
-   **apiKey**：这里是你在云雾中转站申请的专属Key，而不是OpenAI的Key。

### 第四步：从“跑通”到“生产”的细节

1.  **使用流式输出**：生产环境一定不要用上面的同步代码，一定要改成流式响应。SDK自动支持流式（`stream`参数设为`true`），可以让用户获得打字机效果的快感，也能有效降低首字延迟。

2.  **异常处理与重试机制**：网络状况再好，也偶尔会遇到超时。你应该为API调用增加重试机制，比如指数退避重试策略。

3.  **请求限速（Rate Limiting）**：中转站对并发的支持很好（官方标注“无并发限制”），但你的代码本身也要注意，避免在单位时间内发送过多请求到同一个模型。

4.  **API Key轮换与安全**：不要把API Key硬编码在代码里。应该通过环境变量（`System.getenv(“YUNWU_API_KEY”)`）进行读取，并定期轮换。

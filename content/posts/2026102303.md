## 第四步：100% 避开报错的应对策略（逐条核对）

### 常见报错 1：`OpenAIError: The model 'gpt-5.1-codex' does not exist`

**原因：** 你用的云雾 API Key 所在的分组，可能没有开启对 GPT-5.1-Codex 模型的访问权限。

**解决方案：** 登录云雾控制台，进“API Keys”页面，点“编辑”，确认你选的**分组**是“默认（混合）”或者“官转 OpenAI”，这两个分组都支持 GPT-5.1-Codex。如果你选了“限时特价”分组，这个分组主要面向 DeepSeek、Qwen、Gemini 等模型，可能不包括 GPT-5.1-Codex。

### 常见报错 2：`requests.exceptions.ConnectionError`

**原因：** 网络问题，或者你的防火墙、路由器阻止了到云雾服务器的连接。更常见的是有人复制代码时忘记改 `base_url`，还在用 `https://api.openai.com/v1`。

**解决方案：**
1. 逐一核对你代码里的 `base_url`，必须是：`https://www.yunwuai.cc/v1`
2. 如果你在公司网络里，确认公司的代理或防火墙没有屏蔽 `yunwuai.cc` 这个域名。
3. 如果还报错，在终端执行一下 `curl -I https://www.yunwuai.cc`，看看是不是返回 200。

### 常见报错 3：`Incorrect API key provided`

**原因：** 你复制的 API Key 可能有空格、首尾有多余空白、或者 Key 本身已经过期/被删除了。

**解决方案：** 登录云雾控制台，在 API Keys 页面复制 Key 的时候，务必双击全选再 `Ctrl+C`，粘贴到代码里后手动检查前后有没有意外的空格。建议写代码时直接把 Key 设成环境变量：

然后在代码里通过 `os.getenv("YUNWU_API_KEY")` 获取，减少手输错误。

    # 如果数组长度小于等于1，直接返回
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr)//2]  # 选择中间元素作为基准
    left = [x for x in arr if x < pivot]

    return quick_sort(left) + middle + quick_sort(right)

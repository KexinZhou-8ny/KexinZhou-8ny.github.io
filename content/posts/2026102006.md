## 没有代码报错怎么办？常见错误排查

如果运行后出现错误，对照看看：

**错误 401：认证失败**
-   **原因**：API Key 无效或未填写。
-   **解决**：检查 `apiKey` 是否完全复制，注意不要包含多余的空格。重新在后台创建新的 API Key。

**错误 404：模型不存在**
-   **原因**：模型名称写错了。
-   **解决**：检查 `mistral-large-2407` 这个名称是否完全正确。在云雾后台的“模型列表”页面，搜索 “mistral” 确认当前支持的精确模型ID。

**错误 500 或连接超时**
-   **原因**：网络问题或服务端暂时不可用。
-   **解决**：检查你的网络是否正常（可以访问 `www.yunwuai.cc` 网站）。等待几秒后重试。云雾官方宣称可用性高达 99.9%，非常可靠。

**Maven 依赖下载失败**
-   **原因**：Maven 仓库同步问题，或网络防火墙阻挡。
-   **解决**：更换 Maven 镜像源（如阿里云、华为云），或者直接使用 `openai-gpt3-java` 的 jar 包手动导入。

### 异步调用（提升并发吞吐）

把同步调用改为异步，可以并发处理多个用户的请求，提升响应速度。

java
// 使用CompletableFuture
CompletableFuture<ChatCompletionResult> future = service.createChatCompletion(request);
future.thenAccept(result -> {
    // 处理成功结果
    String content = result.getChoices().get(0).getMessage().getContent();
    System.out.println(content);
}).exceptionally(e -> {
    System.err.println("异步调用失败：" + e.getMessage());
    return null;
});

## 接口 401 到底在说什么？

在动手之前，我们得先搞明白 `401` 的本质。它不是说你网络不行，也不是说 API 挂了，它就是在告诉你：**你的请求没有通过身份验证。**

官方文档里会一本正经地告诉你，你需要提供一个有效的 `Authorization` 头。但对于大多数开发者，尤其是国内开发者来说，真正导致 `401` 的，往往不是“没给”，而是“给错了”或者“没法给”。

这里我帮你总结了最常见的几种“踩坑”情况，你可以把这段当作一个自查清单。

### ② 换了 Key 不换地址？照样 401！
很多第三方框架或工具，在集成时默认写死了官方的 API 地址。比如 `https://api.openai.com`。你注册了一个国内的中转 API 服务，拿到了一个新 Key，但在代码里只改了 Key，忘了改请求地址。这时候，你的新 Key 发到了 OpenAI 的服务器上，人家当然不认识，一个 `401` 把你打回原形。

**检查你的 `base_url` 或者 API 请求地址。** 如果你是通过国内平台调用，一定要把 `base_url` 换成该平台提供的服务器地址。比如，如果你使用[云雾ai聚合站](https://www.yunwuai.cc/)，你的请求地址应该是 `https://www.yunwuai.cc/v1`，而不是 `https://api.openai.com/v1`。这一步改对了，99% 的 `401` 都能解决。

这是国内开发者最头疼的痛点。很多时候，你的代码逻辑完全正确，Key 也是刚从官方申请的，但就是发不出请求。这是因为你的服务器或本地网络无法直接访问 OpenAI 等海外 API 服务器。当请求在半路被墙、被阻断时，你的客户端收到的依然是一个 `401` Unauthorized 错误，让你怀疑人生。

### 手把手：用 Java 接入 GPT-5.1 的官方示例

对于企业级 Java 接入，最简单稳定的做法是直接用 OpenAPI SDK 配上云雾的 API 地址。下面展示一段 Java 核心代码示例，用来跑通 GPT-5.1 接入：

java
import com.openai.sdk.OpenAIClient;
import com.openai.sdk.OpenAIClientConfig;
import com.openai.sdk.chat.ChatCompletionRequest;
import com.openai.sdk.chat.ChatCompletionResponse;
import com.openai.sdk.chat.Message;

public class GPT51JavaExample {

    public static void main(String[] args) {
        // 1. 配置 OpenAPI SDK，使用云雾的 API 地址
        OpenAIClientConfig config = OpenAIClientConfig.builder()
                .apiKey("你从云雾申请的 API Key")
                .baseUrl("https://www.yunwuai.cc/v1") 
                .build();

        OpenAIClient client = new OpenAIClient(config);

        // 2. 构建 GPT-5.1 的聊天请求
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("gpt-5.1")  // 模型名称直接替换成 gpt-5.1
                .message(Message.user("你好，GPT-5.1，请回答什么是企业级AI接入？"))
                .maxTokens(1024)
                .temperature(0.7)
                .build();

        // 3. 发送请求并获取流式响应（这里以非流式为例）
        ChatCompletionResponse response = client.chat().create(request);

        // 4. 输出结果
        System.out.println("GPT-5.1 回复：" + response.getChoices().get(0).getMessage().getContent());
    }
}


**代码解释**：
- 通过 Maven 或 Gradle 引入 `com.openai.sdk` 相关依赖，这是 OpenAI 官方 SDK。
- 唯一需要改动的地方就是第 10 行基地址，改成云雾的 API 地址。
- 替换 API key 为你在云雾注册后获得的 Key。

**关键点**：这个方法不改 Java 项目架构，不增加任何依赖问题，唯一动作就是修改 base_url 和 API key。如果是企业已有 GPT-3.5 或 GPT-4 接入代码，只需要将 model 字段改成 `gpt-5.1`，然后 url 换成云雾的链接，就可以直接切到新模型。
